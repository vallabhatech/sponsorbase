# Contributing to this Project

Thank you for your interest in contributing! We welcome all contributions under ELUSOC'26 to help make this project better.

## 🚀 How to Set Up the Project Locally

1. **Fork the Repository:** Click the 'Fork' button at the top right of this page.
2. **Clone Your Fork:**
   ```bash
   git clone git clone https://github.com/YOUR-USERNAME/sponsorbase.git
   ```
3. **Navigate to the Directory:**
   ```bash
   cd sponsorbase
   ```


## 🔄 Contribution Workflow

1. **Create a New Branch:** Always create a descriptive branch for your work.
   ```bash
   git checkout -b feature/your-feature-name
   ```
2. **Make Your Changes:** Implement your feature or fix.

3. **Commit Your Changes:** Write clear, descriptive commit messages.
   ```bash
   git commit -m "--message--"
   ```
4. **Push to Your Fork:**
   ```bash
   git push origin feature/your-feature-name
   ```

## 📋 Issue Reporting Standards

* Check existing issues before creating a new one to avoid duplicates.
* Use a clear, descriptive title.
* Provide steps to reproduce the bug, expected behavior, and screenshots if applicable.

## 🔀 Pull Request Guidelines

* Link your PR to the specific issue it fixes (e.g., `Closes #12`).
* Keep your PR focused on a single change or fix.

## 🛠️ Coding Best Practices

* Write clean, readable, and self-documenting code.
* Add comments for complex logic blocks.
